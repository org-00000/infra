* Pseudo-language
:PROPERTIES:
:ID:       4a1f8c2e-7b3d-4e9a-b5f0-2c6d8e1a4b7f
:END:

A structured pseudo-language used throughout =./doc/= to write specifications and
algorithms. It is language-agnostic: it says /what/ a piece of code must do, not /how/.

Source blocks containing this language are tagged =pseudo=:

#+begin_example
#+begin_src pseudo
  … pseudo-language expressions …
#+end_src
#+end_example

For an AI assistant: when you encounter =#+begin_src pseudo=, parse its content using
the rules in this file. If a construct appears that is not defined here, warn the
user and ask whether it should be added to this document.

Concepts are introduced in dependency order: each section uses only what was defined
before it.

** Comment
:PROPERTIES:
:ID:       2a7f4c1e-8b3d-4e9a-c5f0-3d6b8e2a7f4c
:END:

=%= introduces an inline comment to the end of the line. Comments are ignored when
interpreting the pseudo-language.

#+begin_src pseudo
x y ↦ x + y   % adds two numbers
#+end_src

** # — Qualified aspect access
:PROPERTIES:
:ID:       a2f7c3e1-9b4d-4a0f-d6e8-3c1b7a5f2e9d
:END:

For any concept =T=, =T#name= denotes the aspect =name= of =T=. Inside a definition
block headed by =T=, each listed item =name= is implicitly =T#name=; the prefix is
omitted for readability but always implied.

#+begin_src pseudo
% These two blocks are equivalent:

List(X)
- empty :: …            % shorthand inside the block

List(X)
- List(X)#empty :: …    % fully qualified form
#+end_src

=T#name= may be chained: =E#range#list= means the =list= aspect of the =range= aspect
of =E=.

** Sequential composition
:PROPERTIES:
:ID:       3b8e5f2a-9c4d-4a7b-d1e6-5f2c9b4a8e3d
:END:

Within a branch or function body, expressions written on consecutive lines are
executed top to bottom. Each line may bind names that are visible to all subsequent
lines in the same scope.

#+begin_src pseudo
a :≡ f(x)   % execute f(x), bind result to a
b :≡ g(a)   % execute g(a) — a is in scope
h(b)        % value of the sequence is h(b)'s result
#+end_src

The value of a sequence is the value of its last expression.

** Bindings and destructuring
:PROPERTIES:
:ID:       4c9f6a3b-0d5e-4b8c-e2f1-6a3d7c0f5b9e
:END:

=:≡= is used for both global definitions and local bindings; context determines scope:

- *Global definition* — at the top level of a specification block, =name :≡ expr=
  introduces =name= as a permanent, globally visible definition.

- *Local binding* — inside a function body or Algorithm branch, =name :≡ expr=
  introduces =name= only within the enclosing scope. It is not visible outside.

Tuple destructuring is allowed on the left-hand side:

#+begin_src pseudo
<a, b> :≡ f(x)   % bind a to the first element, b to the second
#+end_src

Local definitions may be recursive: a name bound with =:≡= may appear in its own
definition (including mutual recursion between names in the same scope).

** cond
:PROPERTIES:
:ID:       f6c4a8e3-1d5b-4f0c-c3a2-9e7b5d4c2f8a
:END:

=cond= is a conditional dispatch on boolean conditions (not a value pattern). Branches
are tried top to bottom; the first branch whose condition holds is taken. =_= is the
default (always-true) branch.

#+begin_src pseudo
cond
  condition₁ ↦ expr₁
  condition₂ ↦ expr₂
  _ ↦ default
#+end_src

Branches may be nested:

#+begin_src pseudo
cond
  condition₁ ↦ expr₁
  _ ↦ cond
    condition₂ ↦ expr₂
    _ ↦ expr₃
#+end_src

** match
:PROPERTIES:
:ID:       a7d5b9f4-2e6c-4a1d-d4b3-0f8e6c5a3b9d
:END:

=match= dispatches on the value of an expression using [[ref:e5b3f7d2-0c4a-4e9b-b2f1-8d4e7c2b5a1f][patterns]]. Use it when the
branch depends on the shape or value of a specific term; use =cond= when the branches
depend on arbitrary boolean conditions.

#+begin_src pseudo
match expr
  pattern₁ ↦ expr₁
  pattern₂ ↦ expr₂
  _ ↦ default
#+end_src

** List

List(X)
- empty :: represents an empty list of X.
  - [] :≡ empty
- cons(x list) :: represents the list that starts with x followed by elements of list
  - cons : X List(X) → List(X)
- [a, b…, z] :: shotcut notation for cons(a cons(b …))
- elim(c f) :: represents a function that maps any list to some type of value
  - elim : C (X C → C) → List(X) → C
  - elim(c f) :≡
    1. rec c f list :≡
       - match list
         - [] ↦ c
         - cons x list' ↦ f(x rec(c f list'))
    2. c f ↦ list ↦ rec c f list
- head(list) :: represents the first element of list, if any
  - If l ≡ cons(x _), then: head(l) :≡ x
- map(f lst) :: Given lst : List(X), then it represents a list like lst but with all
  elements mapped according to f
  - map : (X → C) List(X) → List(C)
  - map(f) :≡ elim([] (x list' ↦ cons(f(x) list')))

** Range
:PROPERTIES:
:ID:       bc6d898b-dac0-4ff5-975c-08829f11651a
:END:

Range
- mk(start end) :: represents a set of consecutive n : ℕ such that start ≤ n <
  end.
- start :: represents the starting ℕ of a range.
- end :: represents the end ℕ of a range.
- to(n) :: represents the range mk(0 n)
- list :: represents the unique List(ℕ) associated from some range.
  - list : Range → List(ℕ)
  - list(mk(start end)) :≡ [start, …, end-1]

** Enumerable
:PROPERTIES:
:ID:       735be708-dd63-4b4b-9c24-9575a0a9d638
:END:

Enumerable(X)
- E : Enumerable(X) :: represents a set of element of X indexed by a range
  - E#set : Set(X)
  - ∃ n : ℕ, E#range :≡ Range#to(n)
  - E#index : E#range → E#set
  - E#index is bijective.
- E#at(i) :: represents the element of E#set indexed by i
  - E#at(i) :≡ E#index(i)
- E#list :: represents the list that is isomorphic to E#set according to E#index
  - E#list :≡ map E#index E#range#list
- Since E#set ≅ E#list, then: any operation on E#list may be applied to E#set and
  reciprocally.
