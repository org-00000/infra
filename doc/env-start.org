Represents the development environment.

** env#start
:PROPERTIES:
:ID:       d8afe12e-f756-4c33-a7dd-f380004f91bf
:END:

Premises
- git is installed
- ssh-add is installed
- guix is installed
- qemu-system-x86_64 is installed
- /dev/kvm exists and is accessible by the current user
- env#start

Conclusion
- A development environment has been started.
- The current bash session occurs within a container.
- The session is configured for development.
- It is isolated from the host except for:
  - network access,
  - Guix daemon access to share build capabilities with the host,
  - read access to /etc/guix to sign builds,
  - access to the underlying ssh agent in order to push commits to the remote repository,
  - /tmp is shared with the host,
  - can launch graphical applications (Wayland),
  - Host Guix archive keys are accessible in order to send builds to external resources /e.g./ VPS.
  - Host SSH agent can be used in order to connect to external resources.
- A set of packages are installed.
- Installed packages have a fixed version.
- The README is displayed using the default editor.

** env#session
:PROPERTIES:
:ID:       e529b076-4540-4c26-97f9-d42e4a2ef30e
:END:

Premises
- The [[ref:6612d34b-e5e7-4d42-b1bf-d73bb1ed9906][environment]] is started.
- =source etc/bash-dev-session= has been evaluated.

Conclusion
- Shell conveniences are active:
  - =emacs= opens a new frame in the running Emacs server via =emacsclient -c=, or
    starts a new Emacs instance (which will itself start a server via [[ref:f1e3b5d7-9c2a-4e8f-a4d6-3b0c7e1f5d9a][emacs-server]]).
  - The README is displayed in Emacs at session start.
